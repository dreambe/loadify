# Loadify 能力差距与路线图(对标 JMeter / 阿里云 PTS)

> 诚实的现状盘点:已具备的能力、与成熟压测产品的差距、以及建议的演进顺序。
> 更新于 2026-06。

## 一、已具备(部分体验优于 JMeter 默认)

- **多协议驱动**:HTTP/HTTPS、gRPC(动态调用)、WebSocket、SSE。
- **两种负载模型**:闭环(VU)与开环(到达率/QPS)。JMeter 原生只有线程组闭环,
  开环要装 Throughput Shaping Timer 插件;PTS 两者都有。
- **多步骤场景**:顺序串联(JSON 提取 → `{{变量}}` 传参)与按权重流量配比。
- **逐请求校验点**:状态码 / 响应体 / JSON 路径,8 种比较符,失败原因进实时日志。
- **SLA 阈值**:p50/p90/p95/p99、错误率、QPS,任一不达标判失败。
- **分布式发压**:多 worker 分片 + 精确合并 HdrHistogram;容量感知排队;断连重连恢复。
- **实时 + 历史看板**、结果对比、CSV 导出、定时任务、用例快照。
- **鉴权与多租户基础**:JWT、RBAC、飞书 OAuth、用户管理。
- **可被 Agent 驱动**:MCP server + OpenAPI + CLI。这是 JMeter/PTS 都没有的差异点。

## 二、明显差距(JMeter / PTS 有,Loadify 还没有)

### 数据与参数化
- **CSV 数据源**:目前数据集只支持内联 JSON 数组,没有上传 CSV / 大文件流式读取
  (JMeter CSV Data Set Config、PTS 文件参数)。
- **函数/变量体系**:没有内置函数(随机数、时间戳、UUID、计数器、faker 类),
  JMeter 有 `__Random`、`__time`、`__UUID` 等一大套。
- **关联(correlation)自动化**:串联要手动配提取路径;没有"录制流量自动识别动态参数"。

### 录制与导入
- **没有流量录制 / HAR 导入 / Postman / Swagger 导入**。PTS 支持从 HAR、JMeter
  脚本导入;JMeter 有 HTTP(S) Test Script Recorder。这是新建大批量用例的关键效率项。

### 协议与场景深度
- **缺 TCP/UDP/MQTT/Kafka/JDBC/gRPC流式以外** 的协议(JMeter 插件生态覆盖很广)。
- **思考时间 / 定时器策略单一**:有 think time,但缺高斯/泊松随机定时器、同步定时器
  (集合点,模拟瞬时并发)。
- **没有事务(transaction)概念**:多个请求合并为一个业务事务统计端到端耗时。

### 结果与诊断
- **压测结果不落库明细**:实时日志是采样且不持久化,结束后无法回查每条请求
  (这是之前明确取舍掉的)。PTS 有采样日志留存。
- **没有响应断言之外的检查点报表**:JMeter 有 Assertion Results、聚合报告导出 HTML;
  Loadify 报告较精简,无 PDF/HTML 导出。
- **缺错误归因下钻**:能看到错误率曲线,但点不进去看"这段时间错在哪个接口/状态码分布"。

### 运维与企业能力
- **即时登出/令牌吊销**:JWT 无状态,禁用账号要等过期(已知限制)。
- **没有项目/团队空间隔离**:所有用例/任务在一个全局空间,缺多项目分组、标签、收藏。
- **没有压测预约审批、并发额度配额**(PTS 企业版有)。
- **被压目标的安全围栏**:没有"目标域名白名单 / 禁止压测内网"防护,生产环境有风险。
- **告警**:有 run-finished webhook,但没有压测中实时阈值告警(错误率突增即时通知)。

## 三、建议演进顺序(投入产出比优先)

1. **CSV 数据源上传 + 内置函数**——参数化是真实压测的刚需,门槛最低、收益最高。
2. **HAR / Postman / Swagger 导入**——批量造用例,直接决定能不能规模化使用。
3. **集合点(同步定时器)+ 事务**——衡量瞬时并发与端到端业务耗时,压测专业度的体现。
4. **错误下钻 + HTML/PDF 报告导出**——结果可交付给非技术干系人。
5. **项目空间 / 标签 / 配额**——多团队使用的组织能力。
6. **目标白名单 + 实时告警 + 令牌吊销**——上生产前的安全与运维补齐。

## 四、刻意不做 / 低优先

- JMeter 的 GUI 元件树那套复杂度:Loadify 走"声明式 + 必要时脚本"路线,不复刻。
- 自建协议插件市场:优先把主流协议做深,而非追求数量。
