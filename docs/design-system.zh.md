# Loadify 设计规范(Design System)

> 精密仪器(precision-instrument)风格:深色底、等宽数据字、克制的色彩。
> **第一原则:默认无色。颜色是信号,只在承载含义时出现。**

本文件是约束,也是反思记录。新页面/新组件请遵循;评审时以此为准。

---

## 1. 设计哲学

1. **颜色 = 信号,不是装饰。** 界面默认是单色(文字/`--muted`/边框)。一抹颜色出现,就应该意味着"这里需要你注意"。满屏彩色 = 没有重点。
2. **状态 ≠ 判定。** "运行在哪一步"(生命周期)和"跑得好不好"(判定)是两回事,不能用同一种颜色混为一谈。
   - 一个 **completed(已完成)** 的 run 可能 SLA 不达标——所以"完成"本身**不该是绿色**。绿色留给"明确验证为好"的判定(SLA 通过)。
3. **诚实优先。** 指标会自曝缺陷(协调省略提示、降级告警)。颜色同理:不确定/中性的东西保持中性,别用绿色营造虚假的"一切正常"。
4. **可达性。** 颜色不是唯一信道:状态永远配文字标签(中/英),不靠"看颜色猜含义"。

### 关于"颜色过度"的一次修正(记录在案)
早期把 **completed 染成绿色**,结果:① 把"跑完"误读成"跑得好";② 列表里大多数 run 都 completed,满屏绿稀释了真正该跳出来的**红色失败**。
修正:**completed/stopped → 中性灰**;绿色只给判定类(SLA 通过 / 无回归 / 节点正常);运行中(running)用绿+脉冲,因为它是唯一"正在发生、值得高亮"的状态。

## 2. 颜色语义(唯一事实来源)

色值定义在 `web/app/globals.css` 的 `:root`(深色)与 `html[data-theme="light"]`(浅色)。**永远用变量,不要写死色值。**

| 语义 | 变量 | 何时用 |
|---|---|---|
| 主文字 | `--text` | 正文 |
| 次文字 | `--text-2` | 次要信息 |
| 弱化/中性 | `--muted` | 说明、占位、**中性状态(已完成/已停止)** |
| 好 / 通过 / 健康 | `--green` | **仅判定**:SLA 通过、无回归、节点正常、实时数据流、对比更优、延迟变快 |
| 坏 / 失败 / 异常 | `--red` | 失败、SLA 未达标、回归、节点异常、已断开、错误行 |
| 等待 / 警告 | `--yellow` | 排队/等待中、等待数据、负载 60–85%、降级提示 |
| 品牌 / 可交互 | `--accent`(青) | **仅交互**:链接、激活的导航 tab、主按钮、关键高亮数字。**不表示"成功"** |
| 强调辅色 | `--accent-2`(靛) | 图表第二条线等,极少用 |

> 经验法则:**绿/红 = 判定;黄 = 等待/警告;灰 = 中性/进行中的非判定状态;青 = 可点的东西。** 一个屏幕里出现 3 种以上"信号色"就要警惕过度。

## 3. 状态徽章(`.badge.*`)

| 类 | 颜色 | 含义 |
|---|---|---|
| `.badge.running` | 绿 + 脉冲点 | 运行中(唯一高亮的"正在发生") |
| `.badge.completed` | **中性灰** | 已完成(生命周期,非判定) |
| `.badge.stopped` | 中性灰 | 手动停止 |
| `.badge.queued` / `.pending` | 黄 | 排队 / 等待 |
| `.badge.failed` / `.aborted` | 红 | 失败 / 中止 |
| `.badge.ok` | 绿 | **判定为好**:SLA 通过、无回归、节点正常、账号正常 |

约定:
- **生命周期**(run.status)→ 用对应的 `running/completed/...` 类,经 `statusLabel(t, status)` 出中文。
- **判定**(通过/失败、健康/异常)→ 用 `ok` / `failed`,**不要**复用 `completed`。
- **角色/标签**(admin、协议名等,无好坏)→ 用中性 `completed` 类或纯文字,**不要**上绿/红。

实时状态点(LiveRunChart):`● 实时`=绿、`● 等待数据`=黄、`○ 已断开`=红、`○ 连接中`=灰。

## 4. 排版 / 间距 / 圆角

- 字体:UI 用 `--font-ui`(Space Grotesk + 中文回退);**数据/数字/代码用 `--font-mono`** + `font-variant-numeric: tabular-nums`(对齐)。
- 间距:只用 `--s1..--s7`(4px 基),不要写魔法数字。
- 圆角:面板 ~10px,控件/徽章 ~6–8px。
- 动效:用 `--ease`;**自动刷新的视图不得打断用户**(列表无变化不重渲染、展开的行刷新时冻结)。

---

## 5. 交互准则(评审清单)

- **颜色**:这处颜色承载含义吗?能否改成中性?同一含义全站是否同色?
- **保存即有"已保存"态**:表单提交后回只读展示,编辑是显式动作。
- **输入有约束**:数值给 min/max + 失焦归一,允许中途清空。
- **异步按钮防重复提交**:请求中禁用。
- **空/加载/错误三态齐全**:空有引导 CTA,加载有骨架/提示,失败有可读原因(不留哑状态)。
- **危险操作**:二次确认,危险弹窗默认聚焦"取消"。
- **i18n**:所有用户可见文案走 `t()`;中英 key 必须等量对齐(有脚本校验)。
- **国际化下不混排**:中文界面不出现 `打印 / Print` 这种中英混排。
